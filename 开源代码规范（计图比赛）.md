# 开源代码规范（计图比赛）

本规范用于计图比赛相关的开源代码仓库，目标是：**结构清晰、可复现、易阅读、易运行、易审阅**。

---

## 1. 基本要求

- **必须包含**：`README.md`、`LICENSE`、`.gitignore`
- **不得提交**：数据集原始文件、大模型大权重、训练产物（如未特别说明）
- **所有内容可复现**：从零开始按文档操作，可得到可用结果（允许与最终成绩存在合理差异，但必须说明）

---

## 2. 目录与命名（简单统一即可）

建议目录（可按项目实际调整，但要保持一致）：

- `configs/`：配置文件（训练/评测参数）
- `src/`：核心代码（模型/数据/训练/评测）
- `scripts/`：运行脚本（train/eval/infer）
- `tools/`：数据处理、统计等工具脚本
- `data/`：仅放数据说明（不放大文件）
- `outputs/`：日志、权重、结果（默认不提交）

命名规则：

- 文件/目录：`snake_case`
- 类名：`PascalCase`
- 函数/变量：`snake_case`
- 常量：`UPPER_SNAKE_CASE`

---

## 3. 代码风格（Python）

- 缩进：**4 空格**
- 编码：UTF-8
- 建议行宽：88~100
- 禁止 `except: pass`；异常要给出明确报错信息
- 关键函数必须写清楚注释/文档（至少说明输入输出、shape、关键参数含义）
- 入口脚本中不要写死本机路径（用参数或配置传入）

---

## 4. README 必须包含的内容（最少版）

README 至少包含以下小节（用最短可执行说明即可）：

1. **环境安装**
   - Python 版本
   - 安装命令（`pip install -r requirements.txt` 或等价方式）

2. **数据准备**
   - 数据下载方式/路径结构示例
   - 数据根目录如何配置（如 `--data-root` 或配置文件字段）

3. **训练**
   - 一条可直接运行的命令（含 config）

4. **评测/推理**
   - 一条可直接运行的命令（含 ckpt 路径）

5. **结果说明**
   - 指标名称与计算方式（简述）
   - 若与线上提交/最终成绩有差异，说明原因

---

## 5. 可复现要求（比赛开源重点）

- 必须提供 `--seed`（或配置项）并在代码中统一设置随机种子
- 每次运行建议保存：
  - 实际使用的配置（落盘为 `config.yaml/json`）
  - 运行命令（保存到 `command.txt`）
  - 日志（`train.log` / `eval.log`）
- 训练/评测脚本在缺少文件或路径错误时，必须报清楚缺什么、如何修复

---

## 6. 配置与参数管理（简化版）

- 训练/评测的关键参数（学习率、batch size、epoch、数据路径、模型结构）应来自配置文件或命令行参数
- 命令行参数优先级高于配置文件（如两者同时存在）

---

## 7. `.gitignore` 建议忽略项

至少包含（按项目实际补充）：

- `outputs/`
- `*.log`
- `*.pkl` `*.pt` `*.ckpt` `*.npz` `*.npy`
- `__pycache__/`
- `.pytest_cache/`
- `.DS_Store`

---

## 8. 提交规范（轻量版）

- 提交信息建议使用前缀：
  - `feat:` 新功能
  - `fix:` 修复
  - `docs:` 文档
  - `refactor:` 重构
  - `chore:` 杂项
- 提交前自查：
  - 代码能运行（至少跑通一个小配置/小样本）
  - README 中的命令真实可用
  - 未提交大文件与中间产物

---

## 9. 第三方引用与声明

- 引用/改写第三方代码必须注明来源与许可证（可写在文件头注释或 `NOTICE`）
- 如使用预训练权重/数据/模型，需要说明来源、许可与下载方式
